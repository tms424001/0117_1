工程造价数字化平台 Spec 目录
00_README.md (全局索引与操作规范)
平台全景图：五大模块（采集、资产、质控、计价、估算）的关系说明。

Spec 优先级规则：全局规范 > 模块规范 > 页面规范。

版本管理：v1.0 初始版本，记录后续所有重大变更。

第一部分：全局上位规范 (Global Specs)
规定全站通用的“底线”，确保一致性。

01_Layout_Spec.md (布局规范)
全局框架：顶部固定 Header（5模块切换）、侧边栏 Sidebar（二级菜单）、内容区 Main。

响应式规则：支持 1366px - 4K 分辨率适配。

面板分割：规定“左树右表”及“三栏式工作台”的比例与折叠行为。

02_Interaction_Pattern_Spec.md (交互与组件规范)
三态覆盖：强制要求所有数据容器具备 Loading（骨架屏）、Empty（空数据）、Error（错误重试）状态。

表单与校验：统一必填项标识、防重复提交策略、错误提示位置。

弹窗逻辑：区分 Modal（强制中断交互）与 Drawer（侧边滑出，用于辅助查看/编辑）。

03_Design_Token_Spec.md (视觉变量规范)
色彩体系：主色（造价蓝/专业绿）、状态色（🔴严重、🟡警告、🔵提示）。

间距与栅格：全站统一使用 4px/8px 为步进的间距系统。

第二部分：业务模型与模块 Spec (Module Specs)
详细描述每个模块的业务深度。

04_Data_Collection_Spec.md (数据采集模块 - 核心逻辑)
业务模型：文件上传 → 自动解析 → 个人治理（我的项目/材料/清单）→ 推送审核。

项目元数据：定义业态、地区、建设规模、结构类型等必填字段映射。

材料标准化：展示“原始材料”与“标准库 SML”的映射交互逻辑。

05_Data_Asset_Spec.md (数据资产模块)
企业库结构：材价库、综价库、指标库、案例库的检索与详情展示。

数据商城：资产上架流程、脱敏预览效果。

06_Quality_Control_Spec.md (质控模块)
Issue 逻辑：定义 Issue 的分级、定位跳转、忽略与闭环反馈。

多文件比对：定义 Diff View 的同步滚动、差异高亮算法。

07_Estimation_Spec.md (估算模块)
测算模型：输入参数（设计参数）如何驱动结果计算。

指标联动：如何调用企业指标库作为参考基准。

第三部分：数据模型与接口规范 (Data & API Specs)
08_Data_Model_Spec.md (前端数据实体)
实体定义：Project、Material、BoQ、Index 等 TypeScript 接口定义。

状态管理：规定哪些数据需要跨模块缓存（如标准库、用户信息）。

09_API_Contract_Spec.md (Mock 协议)
响应结构：{ code: 200, data: {}, msg: "" } 统一结构。

分页协议：page, pageSize, total 的命名与交互规范。

第四部分：工程质量与可维护性 (Engineering Specs)
10_Maintainability_Spec.md (代码组织)
目录结构：/pages (页面), /components (业务组件), /ui (基础组件), /hooks (逻辑封装)。

命名约定：PascalCase (组件), camelCase (函数/变量)。

11_Performance_Spec.md (性能与大数据)
大数据加载：规定 1000 行以上表格必须使用虚拟滚动（Virtual Scroll）。

懒加载：按路由进行代码分割（Code Splitting）。
