## 1. 功能概述

### 1.1 功能定位

结算复核是质控模块的**多文件对比能力**之一，以合同价为基准，对施工方提交的结算文件进行全面审核，确保结算金额的准确性和合规性。

### 1.3 复核维度

| 复核维度 | 复核目的 | 依据文件 |
|----------|----------|----------|
| 清单项复核 | 核实结算清单项的有效性 | 合同清单、变更签证 |
| 工程量复核 | 核实结算工程量的准确性 | 合同量、变更量、实测量 |
| 单价复核 | 核实结算单价的合规性 | 合同单价、调价条款 |
| 变更签证复核 | 核实变更签证的完整性和计价正确性 | 签证单、变更令 |
| 费用复核 | 核实各项费用计算的正确性 | 合同约定、费率规定 |

## 2. 结算复核项目管理

### 2.1 项目数据结构

```typescript
// 结算复核项目
interface SettlementReviewProject {
  projectId: string;                    // 项目ID
  projectName: string;                  // 项目名称
  projectCode: string;                  // 项目编号
  createTime: Date;                     // 创建时间
  status: 'draft' | 'in_progress' | 'completed';  // 状态
  
  // 合同文件（基准）
  contractFile: {
    fileId: string;
    fileName: string;
    contractNo: string;                 // 合同编号
    contractDate: Date;                 // 合同签订日期
    contractAmount: number;             // 合同金额
    itemCount: number;                  // 清单项数
    uploadTime: Date;
  };
  
  // 变更签证文件（辅助）
  changeFiles: ChangeFile[];
  
  // 结算文件（对比）
  settlementFile: {
    fileId: string;
    fileName: string;
    settlementAmount: number;           // 结算送审金额
    itemCount: number;                  // 清单项数
    submitDate: Date;                   // 提交日期
    uploadTime: Date;
  };
  
  // 复核轮次
  reviewRounds: ReviewRound[];
  
  // 项目配置
  config: SettlementReviewConfig;
}

// 变更签证文件
interface ChangeFile {
  fileId: string;
  changeType: 'design_change' | 'visa' | 'claim';  // 设计变更/现场签证/索赔
  changeNo: string;                     // 变更编号
  changeDate: Date;                     // 变更日期
  changeAmount: number;                 // 变更金额
  changeItems: ChangeItem[];            // 变更清单项
  status: 'pending' | 'approved' | 'rejected';  // 审批状态
  uploadTime: Date;
}

// 变更清单项
interface ChangeItem {
  itemId: string;
  changeType: 'add' | 'modify' | 'delete';  // 新增/变更/删除
  originalItemCode?: string;            // 原清单编码（变更/删除时）
  newItemCode?: string;                 // 新清单编码（新增/变更时）
  itemName: string;
  quantityChange: number;               // 工程量变化
  priceChange?: number;                 // 单价变化
  amountChange: number;                 // 金额变化
  reason: string;                       // 变更原因
}

// 复核轮次
interface ReviewRound {
  roundId: string;
  roundNumber: number;
  createTime: Date;
  completeTime?: Date;
  status: 'in_progress' | 'completed';
  
  // 复核结果
  reviewResult: SettlementReviewResult;
  
  // 复核意见
  reviewOpinions: ReviewOpinion[];
  
  // 争议项
  disputes: DisputeItem[];
}

// 复核配置
interface SettlementReviewConfig {
  // 工程量偏差阈值
  quantityDeviationThreshold: number;   // 如5%
  
  // 单价偏差阈值
  priceDeviationThreshold: number;      // 如10%
  
  // 材料调差配置
  materialAdjustment: {
    enabled: boolean;
    thresholdPercent: number;           // 调差触发阈值
    adjustmentRange: number;            // 调差比例范围
  };
  
  // 人工调差配置
  laborAdjustment: {
    enabled: boolean;
  };
}
```

## 3. 清单项复核

### 3.1 复核原理

```
┌─────────────────────────────────────────────────────────────────────────┐
│                          清单项复核原理                                  │
├─────────────────────────────────────────────────────────────────────────┤
│                                                                         │
│   清单项来源分类                                                        │
│   ──────────────                                                        │
│                                                                         │
│   ┌─────────────────────────────────────────────────────────────────┐  │
│   │                                                                  │  │
│   │  结算清单项 = 合同清单项 + 新增项 - 删除项 ± 变更项             │  │
│   │                                                                  │  │
│   │  ┌────────────┐                                                  │  │
│   │  │ 合同清单项 │ ← 原合同中的清单项，不可随意增减               │  │
│   │  ├────────────┤                                                  │  │
│   │  │ 新增项     │ ← 必须有变更签证支持                           │  │
│   │  ├────────────┤                                                  │  │
│   │  │ 删除项     │ ← 必须有变更签证支持，或实际未施工             │  │
│   │  ├────────────┤                                                  │  │
│   │  │ 变更项     │ ← 原有项的工程量/单价发生变化                  │  │
│   │  └────────────┘                                                  │  │
│   │                                                                  │  │
│   └─────────────────────────────────────────────────────────────────┘  │
│                                                                         │
│   复核逻辑                                                              │
│   ────────                                                              │
│                                                                         │
│   1. 合同项检查：结算中应包含所有合同清单项                            │
│   2. 新增项检查：每个新增项必须有对应的变更签证                        │
│   3. 删除项检查：删除项需有依据（变更或实际未施工）                    │
│   4. 变更项检查：变更内容需与签证一致                                  │
│                                                                         │
└─────────────────────────────────────────────────────────────────────────┘
```

### 3.2 清单项对比

| 对比结果 | 说明 | 复核要点 |
|----------|------|----------|
| 完全匹配 | 结算项=合同项，编码名称一致 | 检查工程量和单价 |
| 工程量变化 | 同一清单项工程量不同 | 核实变更依据 |
| 单价变化 | 同一清单项单价不同 | 核实调价依据 |
| 新增项 | 结算有、合同无 | 核实变更签证 |
| 缺失项 | 合同有、结算无 | 确认是否实际未施工 |
| 编码变化 | 编码不同但内容相似 | 确认变更原因 |

### 3.3 检查规则

| 规则ID | 规则名称 | 检查逻辑 | 严重程度 |
|--------|----------|----------|:--------:|
| STL-ITM-001 | 合同项缺失 | 合同清单项在结算中不存在 | 🔴 严重 |
| STL-ITM-002 | 新增项无变更依据 | 结算新增项无对应变更签证 | 🔴 严重 |
| STL-ITM-003 | 新增项变更金额不符 | 新增项金额与变更签证不一致 | 🟡 一般 |
| STL-ITM-004 | 删除项无依据 | 合同项被删除但无说明 | 🟡 一般 |
| STL-ITM-005 | 清单编码变更 | 同一项清单编码发生变化 | 🔵 提示 |
| STL-ITM-006 | 清单名称变更 | 同一项清单名称发生变化 | 🔵 提示 |

## 4. 工程量复核

### 4.1 复核原理

```
┌─────────────────────────────────────────────────────────────────────────┐
│                          工程量复核原理                                  │
├─────────────────────────────────────────────────────────────────────────┤
│                                                                         │
│   工程量变化公式                                                        │
│   ──────────────                                                        │
│                                                                         │
│   结算工程量 = 合同工程量 + 变更增加量 - 变更减少量                    │
│                                                                         │
│   示例：                                                                │
│   ┌─────────────────────────────────────────────────────────────────┐  │
│   │ 清单项：矩形柱C30                                                │  │
│   │                                                                  │  │
│   │ 合同工程量：      125.60 m³                                      │  │
│   │ 变更增加量：      +12.30 m³   （设计变更001）                    │  │
│   │ 变更减少量：       -5.20 m³   （设计变更003）                    │  │
│   │ ──────────────────────────────                                   │  │
│   │ 应结算工程量：    132.70 m³                                      │  │
│   │                                                                  │  │
│   │ 结算送审量：      138.50 m³                                      │  │
│   │ ──────────────────────────────                                   │  │
│   │ 差异：            +5.80 m³   （+4.4%）  ⚠️ 需核实               │  │
│   │                                                                  │  │
│   └─────────────────────────────────────────────────────────────────┘  │
│                                                                         │
│   工程量偏差分析                                                        │
│   ──────────────                                                        │
│                                                                         │
│   偏差原因可能包括：                                                    │
│   • 变更签证遗漏：有实际变更但未签证                                   │
│   • 计算错误：施工方工程量计算有误                                     │
│   • 重复计量：同一工程量重复计入                                       │
│   • 计量规则差异：对计量规则理解不同                                   │
│   • 虚报工程量：故意多报工程量                                         │
│                                                                         │
└─────────────────────────────────────────────────────────────────────────┘
```

### 4.2 工程量偏差判定

| 偏差范围 | 判定 | 处理方式 |
|:--------:|------|----------|
| 0% | 完全一致 | 通过 |
| 0%~1% | 微小偏差 | 可能是计算精度差异，视情况通过 |
| 1%~5% | 轻度偏差 | 需核实原因，可能需要现场复核 |
| 5%~10% | 中度偏差 | 必须核实原因，需要详细说明 |
| >10% | 严重偏差 | 重大差异，需要重新计量 |

### 4.3 检查规则

| 规则ID | 规则名称 | 检查逻辑 | 严重程度 |
|--------|----------|----------|:--------:|
| STL-QTY-001 | 工程量严重偏差 | 结算量偏离应结算量>10% | 🔴 严重 |
| STL-QTY-002 | 工程量中度偏差 | 结算量偏离应结算量5%~10% | 🟡 一般 |
| STL-QTY-003 | 工程量轻度偏差 | 结算量偏离应结算量1%~5% | 🔵 提示 |
| STL-QTY-004 | 工程量超合同未变更 | 结算量>合同量但无变更 | 🔴 严重 |
| STL-QTY-005 | 工程量为负 | 结算工程量<0 | 🔴 严重 |
| STL-QTY-006 | 变更量累计错误 | 变更签证工程量累计有误 | 🟡 一般 |

## 5. 单价复核

### 5.1 复核原理

```
┌─────────────────────────────────────────────────────────────────────────┐
│                          单价复核原理                                    │
├─────────────────────────────────────────────────────────────────────────┤
│                                                                         │
│   单价类型                                                              │
│   ────────                                                              │
│                                                                         │
│   ┌─────────────────────────────────────────────────────────────────┐  │
│   │                                                                  │  │
│   │  类型              说明                   复核要点               │  │
│   │  ──────────────────────────────────────────────────────────────│  │
│   │  合同固定单价      合同约定不可调整       结算单价=合同单价     │  │
│   │  可调单价          按合同约定条件调整     核实调价条件和计算    │  │
│   │  暂定单价          待确定的单价           按实际确定价结算      │  │
│   │  新增项单价        变更新增项的单价       按合同约定方式确定    │  │
│   │                                                                  │  │
│   └─────────────────────────────────────────────────────────────────┘  │
│                                                                         │
│   单价调整的常见情形                                                    │
│   ────────────────────                                                  │
│                                                                         │
│   1. 材料调差                                                           │
│   ┌─────────────────────────────────────────────────────────────────┐  │
│   │ 条件：材料价格波动超过合同约定阈值（如±5%）                     │  │
│   │ 公式：调整额 = (结算期材价 - 合同期材价) × 材料用量            │  │
│   │ 核实：材价来源、用量计算、调差范围                              │  │
│   └─────────────────────────────────────────────────────────────────┘  │
│                                                                         │
│   2. 人工调差                                                           │
│   ┌─────────────────────────────────────────────────────────────────┐  │
│   │ 条件：政府发布人工费调整文件                                    │  │
│   │ 公式：按文件规定执行                                            │  │
│   │ 核实：文件适用性、计算正确性                                    │  │
│   └─────────────────────────────────────────────────────────────────┘  │
│                                                                         │
│   3. 新增项定价                                                         │
│   ┌─────────────────────────────────────────────────────────────────┐  │
│   │ 方式1：套用合同相似项单价                                       │  │
│   │ 方式2：按定额计价                                               │  │
│   │ 方式3：协商定价                                                 │  │
│   │ 核实：定价方式是否符合合同约定                                  │  │
│   └─────────────────────────────────────────────────────────────────┘  │
│                                                                         │
└─────────────────────────────────────────────────────────────────────────┘
```

### 5.2 单价对比分析

| 对比情况 | 复核要点 | 常见问题 |
|----------|----------|----------|
| 单价一致 | 确认应为固定单价 | 应调未调 |
| 单价上调 | 核实调价依据 | 无依据调价、调价计算错误 |
| 单价下调 | 确认合理性 | 少见，需关注原因 |
| 新增项单价 | 核实定价方式 | 定价方式不合规、价格偏高 |

### 5.3 检查规则

| 规则ID | 规则名称 | 检查逻辑 | 严重程度 |
|--------|----------|----------|:--------:|
| STL-PRC-001 | 固定单价被修改 | 合同固定单价在结算中被修改 | 🔴 严重 |
| STL-PRC-002 | 单价调整无依据 | 单价调整但无相应变更签证 | 🔴 严重 |
| STL-PRC-003 | 调价计算错误 | 材料/人工调差计算不正确 | 🟡 一般 |
| STL-PRC-004 | 新增项定价不合规 | 新增项定价方式不符合合同约定 | 🟡 一般 |
| STL-PRC-005 | 新增项单价偏高 | 新增项单价明显高于市场价 | 🟡 一般 |
| STL-PRC-006 | 暂定单价未确定 | 暂定单价未按实际确定 | 🔵 提示 |

### 6.2 变更签证与结算对应检查

| 检查项 | 检查逻辑 | 说明 |
|--------|----------|------|
| 签证已纳入结算 | 每个有效签证在结算中有对应体现 | 防止遗漏 |
| 签证金额一致 | 结算中的变更金额与签证一致 | 防止篡改 |
| 无签证的结算变更 | 结算中的变更有对应签证 | 防止虚增 |
| 签证重复计入 | 同一签证未重复计入结算 | 防止重复 |

### 6.3 检查规则

| 规则ID | 规则名称 | 检查逻辑 | 严重程度 |
|--------|----------|----------|:--------:|
| STL-CHG-001 | 结算变更无签证支持 | 结算中的变更项无对应签证 | 🔴 严重 |
| STL-CHG-002 | 签证未纳入结算 | 有效签证未在结算中体现 | 🟡 一般 |
| STL-CHG-003 | 签证金额不一致 | 结算变更金额与签证不符 | 🟡 一般 |
| STL-CHG-004 | 签证重复计入 | 同一签证在结算中重复体现 | 🔴 严重 |
| STL-CHG-005 | 签证有效性存疑 | 签证签字盖章不完整 | 🟡 一般 |
| STL-CHG-006 | 签证超时提交 | 签证提交超过合同约定时限 | 🔵 提示 |

### 12.1 接口列表

| 接口名称 | 方法 | 路径 | 说明 |
|----------|------|------|------|
| 创建复核项目 | POST | `/api/qc/settlement/projects` | 创建结算复核项目 |
