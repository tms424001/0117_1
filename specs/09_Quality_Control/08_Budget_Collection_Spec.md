## 1. 功能概述

### 1.1 功能定位

概算归集是质控模块的**多文件汇总能力**，将多个单项工程概算文件汇总为项目总概算，同时进行一致性检查和指标分析，确保概算编制的完整性和准确性。

### 1.3 归集维度

| 归集维度 | 说明 | 应用场景 |
|----------|------|----------|
| 按单项工程 | 按建筑物/构筑物归集 | 单项概算汇总 |
| 按专业 | 按土建/安装/装饰归集 | 专业分析 |
| 按费用类别 | 按建安/设备/其他归集 | 投资构成分析 |
| 按资金来源 | 按自筹/贷款/拨款归集 | 资金计划 |

## 2. 概算归集项目管理

### 2.1 项目数据结构

```typescript
// 概算归集项目
interface BudgetCollectionProject {
  projectId: string;                    // 项目ID
  projectName: string;                  // 项目名称
  projectCode: string;                  // 项目编号
  createTime: Date;                     // 创建时间
  status: 'draft' | 'in_progress' | 'completed';  // 状态
  
  // 项目基本信息
  projectInfo: {
    projectType: string;                // 项目类型（住宅/商业/工业等）
    projectScale: string;               // 项目规模描述
    totalBuildingArea: number;          // 总建筑面积（m²）
    budgetYear: string;                 // 概算编制年份
    priceBase: string;                  // 价格基准期
  };
  
  // 单项工程概算文件列表
  unitBudgets: UnitBudgetFile[];
  
  // 归集结果
  collectionResult?: CollectionResult;
  
  // 项目配置
  config: BudgetCollectionConfig;
}

// 单项工程概算文件
interface UnitBudgetFile {
  fileId: string;
  unitName: string;                     // 单项工程名称
  unitCode: string;                     // 单项工程编号
  unitType: 'building' | 'structure' | 'outdoor' | 'other';  // 类型
  fileName: string;
  uploadTime: Date;
  
  // 概算金额
  budgetAmount: number;                 // 概算金额
  buildingArea?: number;                // 建筑面积
  
  // 费用构成
  feeComposition: {
    civilWork: number;                  // 土建工程
    decoration: number;                 // 装饰工程
    installation: number;               // 安装工程
    equipment: number;                  // 设备购置
    measures: number;                   // 措施费
    otherFee: number;                   // 其他费用
    taxes: number;                      // 规费税金
  };
  
  // 状态
  status: 'pending' | 'validated' | 'error';
  validationIssues?: Issue[];
}

// 归集结果
interface CollectionResult {
  // 汇总金额
  totalBudget: number;                  // 项目总概算
  
  // 按单项工程汇总
  unitSummary: {
    unitCode: string;
    unitName: string;
    amount: number;
    ratio: number;                      // 占比
  }[];
  
  // 按专业汇总
  professionSummary: {
    profession: string;
    amount: number;
    ratio: number;
  }[];
  
  // 按费用类别汇总
  feeSummary: {
    feeType: string;
    amount: number;
    ratio: number;
  }[];
  
  // 指标分析
  indexAnalysis: {
    unitCost: number;                   // 单方造价
    civilRatio: number;                 // 土建占比
    installRatio: number;               // 安装占比
    equipmentRatio: number;             // 设备占比
  };
  
  // 检查结果
  checkResult: {
    totalIssues: number;
    errorCount: number;
    warningCount: number;
    infoCount: number;
  };
}

// 归集配置
interface BudgetCollectionConfig {
  // 汇总方式
  summaryMode: 'auto' | 'manual';       // 自动/手动
  
  // 检查选项
  checkOptions: {
    consistencyCheck: boolean;          // 一致性检查
    completenessCheck: boolean;         // 完整性检查
    indexCheck: boolean;                // 指标检查
  };
  
  // 报表配置
  reportConfig: {
    format: 'standard' | 'custom';      // 标准格式/自定义
    includeIndex: boolean;              // 包含指标分析
  };
}
```

### 3.2 费用构成汇总

| 费用类别 | 包含内容 | 汇总方式 |
|----------|----------|----------|
| 建筑安装工程费 | 各单项工程土建+安装+装饰 | 逐项累加 |
| 设备及工器具购置费 | 各单项工程设备购置 | 逐项累加 |
| 工程建设其他费 | 建设管理费、勘察设计费等 | 按规定计取或单独编制 |
| 预备费 | 基本预备费+涨价预备费 | 按比例计取 |
| 建设期利息 | 贷款利息 | 按资金计划计算 |
| 流动资金 | 生产性项目需要 | 按行业标准计算 |

### 4.2 检查项目

| 检查类别 | 检查项 | 说明 |
|----------|--------|------|
| **编制依据** | 定额版本 | 各单项工程应使用相同版本定额 |
| | 取费标准 | 工程类别应一致 |
| | 价格基准期 | 材料价格信息期应一致 |
| **费率一致** | 管理费率 | 同类工程管理费率应一致 |
| | 利润率 | 同类工程利润率应一致 |
| | 规费率 | 按统一规定计取 |
| | 税率 | 税率应一致 |
| **价格一致** | 主材价格 | 同种材料价格应一致 |
| | 人工单价 | 人工单价标准应一致 |
| | 机械台班 | 机械台班单价应一致 |

### 4.3 检查规则

| 规则ID | 规则名称 | 检查逻辑 | 严重程度 |
|--------|----------|----------|:--------:|
| BDG-CST-001 | 定额版本不一致 | 各单项概算使用的定额版本不同 | 🔴 严重 |
| BDG-CST-002 | 取费标准不一致 | 同类工程取费标准不同 | 🔴 严重 |
| BDG-CST-003 | 价格基准期不一致 | 价格信息期不同 | 🟡 一般 |
| BDG-CST-004 | 管理费率不一致 | 同类工程管理费率不同 | 🟡 一般 |
| BDG-CST-005 | 利润率不一致 | 同类工程利润率不同 | 🟡 一般 |
| BDG-CST-006 | 税率不一致 | 税率取值不同 | 🔴 严重 |
| BDG-CST-007 | 主材价格不一致 | 同种材料价格偏差>5% | 🟡 一般 |
| BDG-CST-008 | 人工单价不一致 | 人工单价标准不同 | 🟡 一般 |

## 5. 完整性检查

### 5.1 检查原理

```
┌─────────────────────────────────────────────────────────────────────────┐
│                          完整性检查原理                                  │
├─────────────────────────────────────────────────────────────────────────┤
│                                                                         │
```

## 11. API 接口

### 11.1 接口列表

| 接口名称 | 方法 | 路径 | 说明 |
|----------|------|------|------|
| 创建归集项目 | POST | `/api/qc/budget-collection/projects` | 创建概算归集项目 |
| 获取项目列表 | GET | `/api/qc/budget-collection/projects` | 获取归集项目列表 |

| 获取归集结果 | GET | `/api/qc/budget-collection/projects/{projectId}/result` | 获取归集结果 |
| 生成汇总报表 | POST | `/api/qc/budget-collection/projects/{projectId}/report` | 生成汇总报表 |
| 导出总概算 | GET | `/api/qc/budget-collection/projects/{projectId}/export` | 导出总概算文件 |
