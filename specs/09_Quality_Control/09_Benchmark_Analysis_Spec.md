## 1. 功能概述

### 1.1 功能定位

对标分析是质控模块的**多项目横向对比能力**，通过将目标项目与多个标杆项目进行指标对比，发现造价偏差、识别优化空间、支撑决策分析。

### 1.3 对标维度

| 对标维度 | 核心指标 | 分析目的 |
|----------|----------|----------|
| **经济指标** | 单方造价、总造价、分部造价 | 造价水平对比 |
| **技术指标** | 含钢量、含混凝土量、窗地比 | 设计经济性对比 |
| **结构指标** | 专业占比、人材机占比、措施费占比 | 造价构成对比 |
| **效率指标** | 工期造价比、变更率、结算增长率 | 管理效率对比 |

## 2. 对标分析项目管理

### 2.1 项目数据结构

```typescript
// 对标分析项目
interface BenchmarkProject {
  projectId: string;                    // 项目ID
  projectName: string;                  // 项目名称
  analysisType: 'single' | 'multi' | 'deep';  // 分析模式
  createTime: Date;
  status: 'draft' | 'analyzing' | 'completed';
  
  // 目标项目
  targetProject: ProjectFile;
  
  // 标杆项目（可以是库中项目或上传项目）
  benchmarkProjects: BenchmarkFile[];
  
  // 筛选条件
  filterCriteria: FilterCriteria;
  
  // 分析配置
  analysisConfig: AnalysisConfig;
  
  // 分析结果
  analysisResult?: BenchmarkResult;
}

// 项目文件
interface ProjectFile {
  fileId: string;
  projectName: string;
  projectCode: string;
  
  // 项目属性
  attributes: {
    projectType: string;              // 项目类型（住宅/商业/工业等）
    structureType: string;            // 结构形式
    buildingArea: number;             // 建筑面积
    floors: number;                   // 层数
    region: string;                   // 地区
    completionYear: number;           // 竣工年份
    priceBase: string;                // 价格基准期
  };
  
  // 造价数据
  costData: {
    totalCost: number;                // 总造价
    unitCost: number;                 // 单方造价
    civilCost: number;                // 土建造价
    installCost: number;              // 安装造价
    decorationCost: number;           // 装饰造价
  };
  
  // 技术指标
  technicalIndex: {
    steelContent: number;             // 含钢量(kg/m²)
    concreteContent: number;          // 含混凝土量(m³/m²)
    formworkContent: number;          // 含模板量(m²/m²)
    windowFloorRatio: number;         // 窗地比
  };
  
  // 详细数据（分部分项）
  detailData?: DetailData;
}

// 筛选条件
interface FilterCriteria {
  projectTypes: string[];             // 项目类型
  structureTypes: string[];           // 结构形式
  areaRange: { min: number; max: number };  // 面积范围
  floorRange: { min: number; max: number }; // 层数范围
  regions: string[];                  // 地区
  yearRange: { min: number; max: number };  // 年份范围
  priceAdjustment: boolean;           // 是否进行价格调整
}

// 分析配置
interface AnalysisConfig {
  // 对标维度
  dimensions: ('economic' | 'technical' | 'structural' | 'efficiency')[];
  
  // 对标深度
  depth: 'project' | 'division' | 'item' | 'material';
  
  // 偏差阈值
  deviationThreshold: {
    warning: number;                  // 告警阈值（如15%）
    significant: number;              // 显著阈值（如30%）
  };
  
  // 价格调整
  priceAdjustment: {
    enabled: boolean;
    baseYear: number;                 // 基准年份
    adjustmentFactors: { year: number; factor: number }[];
  };
}

// 对标结果
interface BenchmarkResult {
  // 概要
  summary: {
    targetUnitCost: number;
    benchmarkAvgUnitCost: number;
    deviation: number;
    ranking: number;
    totalBenchmarks: number;
  };
  
  // 经济指标对比
  economicComparison: EconomicComparison;
  
  // 技术指标对比
  technicalComparison: TechnicalComparison;
  
  // 结构指标对比
  structuralComparison: StructuralComparison;
  
  // 详细对比（按深度）
  detailComparison?: DetailComparison;
  
  // 分析结论
  conclusions: Conclusion[];
  
  // 优化建议
  recommendations: Recommendation[];
}
```

### 3.2 价格调整

不同时期的项目需要进行价格调整后才能对比：

```typescript
// 价格调整公式
adjustedCost = originalCost × adjustmentFactor

// 调整系数示例（以2025年为基准）
adjustmentFactors = {
  2020: 1.25,  // 2020年项目乘以1.25调整到2025年价格水平
  2021: 1.20,
  2022: 1.15,
  2023: 1.08,
  2024: 1.03,
  2025: 1.00
}
```

| 指标名称 | 计算公式 | 对比意义 |

```

│                        技术指标对标原理                                  │

│                                                                         │
│   技术指标反映设计的经济性                                              │
│   ────────────────────────                                              │
│                                                                         │
│   ┌─────────────────────────────────────────────────────────────────┐  │
│   │                                                                  │  │
│   │  指标              目标项目    标杆均值    偏差      判定        │  │

│   │  含钢量(kg/m²)        58         52       +11.5%    ⚠️ 偏高    │  │

│   │                                                                  │  │

│                                                                         │
│   技术指标偏差的影响分析                                                │
│   ────────────────────────                                              │
│                                                                         │
│   示例：含钢量偏高11.5%的影响                                          │
│   ┌─────────────────────────────────────────────────────────────────┐  │
│   │                                                                  │  │
│   │  建筑面积：28,400 m²                                            │  │
│   │  目标含钢量：58 kg/m²                                           │  │
│   │  标杆含钢量：52 kg/m²                                           │  │

│   │                                                                  │  │

│   └─────────────────────────────────────────────────────────────────┘  │
│                                                                         │
└─────────────────────────────────────────────────────────────────────────┘
```

### 4.2 对标指标

| 指标名称 | 计算公式 | 参考范围 | 影响因素 |
|----------|----------|:--------:|----------|
| 含钢量 | 钢筋总量/建筑面积 | 45~75 kg/m² | 结构形式、层数、抗震 |
| 含混凝土量 | 混凝土总量/建筑面积 | 0.35~0.55 m³/m² | 结构形式、层数 |
| 含模板量 | 模板总量/建筑面积 | 1.5~2.5 m²/m² | 结构复杂度 |
| 窗地比 | 窗面积/建筑面积 | 0.15~0.25 | 建筑类型、节能要求 |
| 体形系数 | 外表面积/体积 | 0.25~0.40 | 建筑形态 |

### 7.2 标杆匹配算法

```typescript
// 标杆匹配算法
interface BenchmarkMatcher {
  // 匹配标杆项目
  matchBenchmarks(
    target: ProjectFile,
    criteria: FilterCriteria,
    library: BenchmarkLibrary
  ): MatchedBenchmark[];
}

// 匹配结果
interface MatchedBenchmark {
  project: ProjectFile;
  matchScore: number;           // 匹配度分数（0-100）
  matchDetails: {
    typeMatch: boolean;         // 类型匹配
    structureMatch: boolean;    // 结构匹配
    areaMatch: boolean;         // 面积匹配
    regionMatch: boolean;       // 地区匹配
    yearMatch: boolean;         // 年份匹配
  };
}

// 匹配度计算
function calculateMatchScore(target: ProjectFile, benchmark: ProjectFile): number {
  let score = 0;
  
  // 类型完全匹配：+30分
  if (target.attributes.projectType === benchmark.attributes.projectType) {
    score += 30;
  }
  
  // 结构形式匹配：+20分
  if (target.attributes.structureType === benchmark.attributes.structureType) {
    score += 20;
  }
  
  // 面积相近（±30%）：+20分
  const areaRatio = benchmark.attributes.buildingArea / target.attributes.buildingArea;
  if (areaRatio >= 0.7 && areaRatio <= 1.3) {
    score += 20;
  }
  
  // 同地区：+15分
  if (target.attributes.region === benchmark.attributes.region) {
    score += 15;
  }
  
  // 近3年：+15分
  const yearDiff = Math.abs(target.attributes.completionYear - benchmark.attributes.completionYear);
  if (yearDiff <= 3) {
    score += 15;
  }
  
  return score;
}
```

## 附录 B：标杆匹配评分标准

| 匹配维度 | 满分 | 评分规则 |
|----------|:----:|----------|
| 项目类型 | 30分 | 完全匹配得30分，不匹配得0分 |
| 结构形式 | 20分 | 完全匹配得20分，相近得10分 |
| 建筑面积 | 20分 | ±30%以内得20分，±50%以内得10分 |
| 地区 | 15分 | 同市得15分，同省得10分，其他得5分 |
| 年份 | 15分 | 3年内得15分，5年内得10分，其他得5分 |

**匹配度等级**：
- 90分以上：高度匹配
- 70-89分：中度匹配
- 50-69分：低度匹配
- 50分以下：不建议对标
