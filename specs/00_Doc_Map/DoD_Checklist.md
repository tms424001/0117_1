# DoD Checklist（通用完成定义模板）

> v0.9  
> 使用：每份页面类 Spec 交付前逐条打勾

## 1 文档头
- [ ] 标题/版本/日期/模块/Owner
- [ ] 目标用户与目标场景
- [ ] 在主线作业流中的位置

## 2 页面与IA
- [ ] 页面列表+路由
- [ ] 入口与返回路径
- [ ] 至少1张核心线框图（ASCII即可）

## 3 字段表与DTO
- [ ] 列表字段表（列名/类型/必填/校验/示例）
- [ ] 详情字段表（同上）
- [ ] DTO（List/Detail/Edit）

## 4 交互与状态
- [ ] 3~5条关键用户路径（作业流）
- [ ] 批量操作（如有）
- [ ] Loading/Empty/Error 三态

## 5 校验与提示
- [ ] 阻断 vs 警告分级
- [ ] 定位修复入口（从问题→字段/行）
- [ ] 低置信/缺字段的待确认机制（如适用）

## 6 API
- [ ] API清单
- [ ] 至少3个关键接口给请求/响应示例
- [ ] 错误码与处理策略（重试/回滚/提示）

## 7 权限与审计
- [ ] 角色权限（页面/按钮）
- [ ] 操作日志至少3类（提交/发布/锁定等）

## 8 性能
- [ ] 核心页面性能目标
- [ ] 大数据量策略（分页/虚拟滚动）

## 9 验收用例（Given/When/Then）
- [ ] ≥5条（成功/失败/边界/并发/冻结或回滚）
